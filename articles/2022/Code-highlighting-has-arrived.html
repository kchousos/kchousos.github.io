Feature: Code highlighting has arrived!

Since this site has to do with tech and programming, it will naturally have code blocks. With my previous configuration, exported code blocks from org-mode worked just fine, except with one caveat: they weren't highlighted.

Although syntax highlighting and if it is needed can be a controversial topic where views differ, I thought that it would be a nice QOL improvement.

Normally, the htmlize emacs package would do the trick. But, as many other have pointed out1, 2, 3, it doesn't work when emacs is run in -batch mode, which is necessary for the auto-publication of this site.

So my solution was to "borrow" this snippet from this reddit thread,Which makes the exported html Highlight.js-compatible. 

(defun my/org-html-src-block (html)
  "Modify the output of org-html-src-block for highlight.js"
  (replace-regexp-in-string
   "</pre>" "</code></pre>"
   (replace-regexp-in-string
    "<pre class=\"src src-\\(.*\\)\">"
    "<pre><code class=\"\\1\">"
    html)))

(advice-add 'org-html-src-block :filter-return #'my/org-html-src-block)

Now, with the following html code added to my automatic build script: 

<!-- HTML HEAD -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.6.0/styles/base16/papercolor-light.min.css" media="screen"/>
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.6.0/styles/base16/papercolor-dark.min.css" media="screen and (prefers-color-scheme: dark)"/>
...
<!-- HTML BODY -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.6.0/highlight.min.js"></script>
<script>hljs.highlightAll();</script>

The code blocks are highlighted based on the language's syntax. Also, in the same way the main css is changed based on whether dark-mode is enabled, the highlight.js theme is changed as well.

Caveats

The only downside of this approach is that the highlighting becomes depended to whether highlight.js supports the language (e.g., emacs-lisp is not supported as the first code block indicates). Edit: you can get lisp syntax highlighting by including the language seperately, since it's not in the common set of the main link. Lisp is added by the following line: 

<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.6.0/languages/lisp.min.js"></script>
